                            # 2.1 消息队列

                            当一个请求有很多步骤，并且步骤之间是顺序执行的时候，总耗时是每一个分步的耗时之和。 比如一个支付业务，需要完成支付，购物车更新，订单更新，库存更新，短信提醒等很多业务，如果同步进行，互相依赖耦合严重。如果支付完成之后，告诉一个中介，这个订单付款成功了，然后就结束。由这个中介去通知各个第三方或者其他模块完成各自任务。这个就是消息队列的目的。

                            # 2.2 RabbitMQ

                            RabbitMQ 是一个由 Erlang 语言开发的 AMQP 的开源实现。

                            AMQP ：Advanced Message Queue，高级消息队列协议。它是应用层协议的一个开放标准，为面向消息的中间件设计，基于此协议的客户端与消息中间件可传递消息，并不受产品、开发语言等条件的限制。

                            # 2.3 框架模型

                            1. Publisher: 消息的发布者，给消息队列发布消息
                            2. Queue: 消息队列，保存发布者发布过来的消息。一个消息可以被发布到一个或者多个队列。
                            3. Exchange: 交换器，连接Publisher和Queue
                            4. Cconsumer: 消息的订阅者，可以在Queue中获取对应的消息

                            # 2.4 消息类型

                            1. 发布者发布到Queue，订阅者直接在Queue中获取消息
                            2. 发布者发布到Queue，多个订阅者直接在Queue中获取消息
                            3. fanout: 发布者发布到Exchange，Exchange再传到队列，消息会广播到所有订阅者，同一个消息所有订阅者都能收到。
                            4. direct: 发布者发布到Exchange(指明发布到什么交换器的队列)，Exchange再传到队列，订阅者只能收到对应消息
                            5. topic: 相似于direct，只是topic支持通配符路径匹配，direct是完全匹配

public class Send {
    private final static String QUEUE_NAME="hello";
    public static void main(String[] args) {
        ConnectionFactory factory = new ConnectionFactory();
        factory.setHost("localhost");
        factory.setPassword("123456");
        factory.setUsername("root");
        try {
            Connection connection = factory.newConnection();
            Channel channel = connection.createChannel();

            channel.queueDeclare(QUEUE_NAME, false, false, false, null);
            String message = "Hello Rabbit!";
            channel.basicPublish("", QUEUE_NAME, null, message.getBytes());
            System.out.println(" [x] Sent '" + message + "'");

        } catch (IOException e) {
            e.printStackTrace();
        } catch (TimeoutException e) {
            e.printStackTrace();
        }
    }
}

            e.printStackTrace();

            e.printStackTrace();

    }
}

                            # 3.2.2 在队列中读取数据

                            public class Recv {
    private final static String QUEUE_NAME = "hello";

    public static void main(String[] args) throws Exception {
        ConnectionFactory factory = new ConnectionFactory();
        factory.setHost("127.0.0.1");
        factory.setUsername("root");
        factory.setPassword("123456");

        Connection connection = factory.newConnection();
        Channel channel = connection.createChannel();
        channel.basicQos(1);//每次处理1个message
        channel.queueDeclare(QUEUE_NAME,false,false,false,null);
        System.out.println(" [*] Waiting for messages. To exit press CTRL+C");


        DeliverCallback deliverCallback = (consumerTag, delivery) -> {
            String message = new String(delivery.getBody(), "UTF-8");
            System.out.println(System.currentTimeMillis());
            System.out.println(" [x] Received '" + message + "'");
            try {
                doWork(message);
            } catch (Exception e){
                e.printStackTrace();
            }finally {
                System.out.println(" [x] Done");
                channel.basicAck(delivery.getEnvelope().getDeliveryTag(),false);
            }

        };
        boolean autoAck = false; // acknowledgment is covered below
        channel.basicConsume(QUEUE_NAME, autoAck, deliverCallback, consumerTag -> { });
    }

    private static void doWork(String task) throws InterruptedException {
        for (char ch: task.toCharArray()) {
            if (ch == '.') Thread.sleep(1000);
        }
    }
}

                            # 3.3 发布订阅模型open in new window

                            消息发布者发布消息到交换器
                            交换器连接队列
                            消息订阅者在对应的队列取数据
                            多个消息订阅者可以同时接收相同的消息 交换器类型是fanout，类似于广播，所有绑定了的订阅者都能接受到消息。

                            # 3.3.1 发送端代码

                            public class Send {
    private final static String EXCHANGE_NAME="logs";
    private static final ConnectionFactory factory = new ConnectionFactory();
    static{
        factory.setHost("localhost");
        factory.setPassword("123456");
        factory.setUsername("root");
    }
    public static void main(String[] args) {
        Scanner sc=new Scanner(System.in);

        while (!Objects.equals(message = sc.nextLine(), "-1")){
            sendMessage(message);
        }
    }
    public static void sendMessage(String message){
        try {
            Connection connection = factory.newConnection();
            Channel channel = connection.createChannel();
            channel.exchangeDeclare(EXCHANGE_NAME,"fanout");
            message=String.join(" ",message);

            e.printStackTrace();

        }

}

                            # 3.4 路由模式open in new window

                            交换机类型是direct,如图C2订阅了info error warning 三种消息， C1订阅了error 一种消息
                            当发布error消息的时候，C1,C2都可以接收到,当发布info warning消息时候，只有C2可以接受到,当发布其他类型，比如debug类型时候，C1和C2都接收不到。

                            # 3.4.1 发送端代码

                            public class Send {
    private final static String EXCHANGE_NAME="direct_logs";
    private static final ConnectionFactory factory = new ConnectionFactory();
    static{
        factory.setHost("localhost");
        factory.setPassword("123456");
        factory.setUsername("root");
    }
    public static void main(String[] args) {
        Scanner sc=new Scanner(System.in);
        String message ;
        while (!Objects.equals(message = sc.nextLine(), "-1")){
            String[] split = message.split(" ");
            sendMessage(split[0],split[1]);
        }
    }
    public static void sendMessage(String message,String severity){
        try {
            Connection connection = factory.newConnection();
            Channel channel = connection.createChannel();
            channel.exchangeDeclare(EXCHANGE_NAME,"direct");
            message=String.join(" ",message);
            channel.basicPublish(EXCHANGE_NAME,severity, null, message.getBytes(StandardCharsets.UTF_8));
            System.out.println(" [x] Sent '" + severity+"':'"+message + "'");
            channel.close();
            connection.close();
        } catch (IOException e) {
            e.printStackTrace();
        } catch (TimeoutException e) {
            e.printStackTrace();
        }
    }
}
